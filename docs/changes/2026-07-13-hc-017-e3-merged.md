## Implemented lifecycle

### Correction

- confirmed source/value fields are never updated in place;
- correction inserts a new immutable observation snapshot;
- predecessor/successor links preserve the complete chain;
- exact OCR/source snapshots are copied to the replacement;
- active consent, optimistic version and idempotency are required;
- source, unit/range, date, profile and structured-record acknowledgements must be repeated;
- `not_present` requires a separate fresh profile-assignment acknowledgement;
- application execute is revoked from the older acknowledgement-free SQL signature.

### Void

- owner/edit can explicitly remove an active observation from active use;
- a bounded reason and optimistic version are required;
- source/value/provenance remain immutable;
- void remains available after consent withdrawal;
- voided observations are excluded from normal view/analyze reads.

### Permanent erasure

- only the profile owner can permanently erase;
- irreversible confirmation and optimistic version are required;
- the complete connected correction chain is removed atomically;
- source snapshots, originating Lab drafts and value-bearing Lab audit rows are removed;
- only a generic content-free erasure tombstone remains;
- erasure remains available after consent withdrawal.

## Document-linked protection

- owner-only document Lab erasure marks the source document `deletion_pending`;
- all Lab observations/drafts derived from that document are removed atomically;
- an independent RLS helper hides Lab rows for any `deletion_pending` or erased source document;
- a protected transition trigger locks related drafts/observations in deterministic order with `NOWAIT`;
- concurrent lifecycle changes return controlled `HC409` instead of deadlock or post-erasure orphan creation;
- observation-chain erasure rechecks and stabilizes chain membership before deletion.

## Privacy hardening

The E2 source-snapshot policy was too broad. E3 now enforces:

```text
active structured observation: owner/edit/view/analyze
superseded or voided history: owner/edit only
OCR reviewed-text source snapshots: owner/edit only
```
