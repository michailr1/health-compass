The architecture defines the secure path:

```text
Upload analysis
→ quarantine
→ malware and format validation
→ OCR processing
→ human review
→ explicit confirmation
→ confirmed lab observations
→ metric dynamics
```

Key decisions:

- PDF/JPEG/PNG only in the first slice;
- server-side streaming limits: 20 MiB, 50 PDF pages, 25 megapixels;
- quarantine-first and scanner fail closed;
- private storage keys generated by the server;
- raw documents are unavailable to the `analyze` permission;
- a separate restricted worker role processes jobs;
- OCR candidates remain `needs_review` until explicit confirmation;
- no silent unit conversion or source-text replacement;
- owner-only permanent erasure covers raw, derived, OCR and sole-provenance confirmed data;
- document contents and medical values are forbidden in ordinary logs;
- external OCR/LLM requires separate consent and provider approval.

The first implementation slice is deliberately smaller than OCR:

```text
HC-017 Slice B — Secure Document Intake Foundation
```

It will add schema, RLS, private storage abstraction, streamed quarantine upload and minimal status UI for demo/test documents. OCR, real lab facts and production rollout remain out of scope.
