# 05 — Продуктовая и UX-спецификация Health Compass

Версия: 1.0 · Дата: 2026-07-08 · Wireframes: `05-portal-wireframes.pdf`
Связано: `01-health-compass-master-plan.md` (функциональные направления), `12-product-strategy-and-monetization.md`.

## 1. Информационная архитектура (sitemap)

```
/login  /auth/email  /onboarding
/w/:workspace
  /profiles (выбор human|pet)
  /p/:profile            — human
    dashboard | timeline | labs | documents | measurements
    symptoms | conditions | medications | sleep | activity
    nutrition | habits | genetics | devices | goals | reports | assistant
  /pet/:pet              — pet
    dashboard | timeline | labs | visits | medications | vaccinations
    feeding | activity | behavior | devices | care-tasks | assistant
  /family                — семейный dashboard (люди + питомцы)
  /settings              — доступы, уведомления, интеграции, аудит, экспорт, тариф
```

Правило навигации: контекст «чей это профиль» всегда виден (аватар+имя+тип human/pet в шапке); переключение профиля — явное действие; human- и pet-разделы визуально маркированы (иконка вида животного, отличный акцентный оттенок), чтобы исключить перепутывание.

## 2. Ключевые user flows

1. **Первый вход:** Google/magic link → onboarding (имя профиля, дата рождения, пол — опционально; согласия) → пустой dashboard с тремя CTA: «Загрузить анализ», «Подключить Oura», «Добавить измерение». Никаких демо-данных в реальном профиле (RISK-006); демо — отдельный явный «пример-профиль» с плашкой.
2. **Загрузка анализа:** drag&drop → статус «обрабатывается» → экран подтверждения извлечения (слева скан с подсветкой, справа таблица показателей, confidence-бейджи; низкая уверенность = поле требует правки) → подтвердить → показатели попадают в динамику и timeline. Time-to-first-value: < 5 минут от регистрации до первого подтверждённого анализа — это activation event.
3. **Подготовка к врачу:** выбор периода и тем → отчёт: изменившиеся показатели, события, лекарства, вопросы врачу (редактируемые) → экспорт PDF/печать.
4. **Шаринг (PHASE-10):** owner → «Пригласить» → email + уровень (view/analyze/edit) → invitation → принятие с существующим или новым аккаунтом.
5. **Pet-визит:** карточка питомца → «Визит к ветеринару» → загрузка заключения/анализов → vet timeline; красные флаги (отказ от воды, затруднённое дыхание и т.д.) в любом вводе симптомов ведут на экран «немедленно к врачу» без AI-рассуждений.

## 3. Dashboard

Отвечает на вопросы: что изменилось / что требует внимания / что сделать сегодня / какие данные устарели / что не подтверждено / какие интеграции сломаны / прогресс целей. Композиция: (1) строка статуса интеграций и неподтверждённых документов; (2) карточки «Внимание» (жёлтый/красный — только по фактам: просроченный курс, вышедший за референс показатель с датой и источником); (3) тренды (2–4 графика Recharts); (4) задачи на сегодня. Режимы (пресеты набора карточек): простой / расширенный / медицинский / спортивный / семейный / pet / послеоперационный (pet). Семейный dashboard: строка на человека/питомца → статус, ближайшая задача, алерты.

## 4. Экран анализов (labs)

Таблица показателей × время; ячейки с флагом вне референса; клик → график динамики с полосой референса (референс — по полу/возрасту/лаборатории, источник нормы подписан); переключатель единиц (исходные/СИ); «связанные показатели» (панели); кнопка «объяснить» → AI-ассистент с обязательными цитатами на конкретные результаты и дисклеймером «не диагноз». Для pet-анализов — референсы вида/породы, человеческие нормы недоступны даже в UI-выборе.

## 5. AI-ассистент (UX-правила)

Каждый ответ: блок «на чём основано» (даты, документы — кликабельные), маркировка Факт / Интерпретация / Рекомендация тремя визуально разными стилями; красные флаги — отдельная плашка сверху; кнопки «неверно/полезно» (ai_feedback). Ассистент не может изменять данные; предлагает действие → пользователь подтверждает явным кликом. В pet-контексте шапка чата показывает вид/породу/вес — данные, на которых строится ответ.

## 6. Дизайн-система

Токены: базовая сетка 4px; радиус 12 (карточки) / 8 (контролы); шрифт Inter (кириллица), моно — JetBrains Mono для значений. Палитра (спокойная, не «тревожная»): фон #F8FAFC/тёмный #0F172A; primary #0E7490 (teal — human), pet-акцент #7C5CBF (фиолетовый), success #15803D, warning #B45309, danger #B91C1C (только для подтверждённых фактов, не для гипотез). Статусы: confirmed (сплошной), needs_review (пунктир), ai-hypothesis (фиолетовый контур + подпись «гипотеза»). Графики: одна метрика = один цвет всюду; референсные зоны — заливка 8% opacity. Timeline: вертикальная, группировка по месяцам, иконки типов событий, фильтры по типу/источнику. Dark mode: обязателен (медицинские записи читают ночью). Accessibility: контраст AA, полная клавиатурная навигация, aria для графиков (таблица-дублёр).

Компонентная карта (MVP): AppShell, ProfileSwitcher, StatCard, TrendChart, ReferenceBandChart, TimelineItem, DocumentCard, ExtractReviewTable, ConfidenceBadge, FactBadge/InterpretationBadge/RecommendationBadge, RedFlagBanner, EmptyState, ConsentDialog, ShareDialog, PetHeader.

## 7. Мобильная навигация

Bottom-bar: Dashboard / Timeline / «+» (быстрый ввод: фото анализа, вес, симптом) / Ассистент / Профиль. Быстрый ввод — главный мобильный сценарий.

## 8. Критерии приёмки UX (MVP)

1. Новый пользователь без подсказок доходит от логина до подтверждённого анализа ≤ 5 мин. 2. Ни один AI-текст не отображается без блока источников. 3. Human/pet контексты нельзя перепутать (различие проверяется юзабилити-тестом «в каком профиле вы находитесь?» ≥ 95% верных ответов). 4. Пустые состояния везде содержательны (что сделать дальше). 5. Отчёт врачу печатается на A4 без обрезаний.
