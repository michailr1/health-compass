# ADR-001 — Modular monolith вместо микросервисов

Статус: Принято · Дата: 2026-07-08

Контекст: соло-владелец, один VPS, MVP. Решение: один FastAPI-процесс с модульной структурой (identity/tenancy/documents/labs/wearables/ai/...) + отдельные воркер-процессы. Микросервисы не вводятся без доказанной необходимости (нагрузка, команда). Последствия: простые деплой и отладка; дисциплина границ модулей обязательна. Пересмотр: при >3 инженерах или узком месте конкретного модуля.
