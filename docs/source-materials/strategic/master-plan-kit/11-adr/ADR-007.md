# ADR-007 — RLS: FORCE + SECURITY DEFINER функции во владении BYPASSRLS-роли

Статус: Принято · Дата: 2026-07-08

Инцидент 54001 показал: FORCE RLS применяет политики к владельцу-migrator, чьи definer-функции сами читают защищаемые таблицы → рекурсия. Решение (миграции 0020/0021): роль health_compass_rls_definer (NOLOGIN, BYPASSRLS, минимальные SELECT-права), функции SECURITY DEFINER + SET search_path='' + SET row_security=off + полностью квалифицированные имена + REVOKE FROM PUBLIC + EXECUTE только app-роли. Инварианты: новая таблица = ENABLE+FORCE+политики+тест в одном PR; каждая политика проходит регресс-тест на рекурсию. Альтернатива (нерекурсивные политики без definer) допустима для некритичных таблиц.
