# ADR-002 — Аутентификация: прямой Google OIDC + Email Magic Links, без внешнего IAM

Статус: Принято · Дата: 2026-07-08

Authentik/Keycloak отвергнуты: лишний stateful-сервис, поверхность атаки и операционная нагрузка для соло-проекта. Прямой OIDC с PKCE+state+nonce, magic links c хэшированными одноразовыми токенами и rate-limit — реализовано в ветке. Mock-auth в production запрещён конфигом (validate_production). Пересмотр: enterprise-SSO при B2B.
