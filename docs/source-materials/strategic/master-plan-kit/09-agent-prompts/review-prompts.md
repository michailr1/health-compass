# Промты: независимый Reviewer

## Шаблон

```
Ты — независимый reviewer этапа <PHASE-xx> проекта Health Compass. Тебе даны: diff/PR, 02-technical-architecture.md
(§2.3 RISK-регистр, §3.2 инварианты), 07-security-and-threat-model.md, соответствующие ADR и war games (08).
Проверь по чек-листам ниже. Формат ответа: вердикт APPROVE / REQUEST CHANGES + таблица замечаний
(severity: blocker/major/minor; файл; описание; предложение). Не предлагай рефакторинг вне рамок этапа.
```

## Чек-листы

Architecture: границы модулей не нарушены; нет новых сервисов без ADR; одна транзакция на запрос сохранена; очередь/хранилище соответствуют ADR-009/016.

Security/RLS: каждая новая таблица — ENABLE+FORCE+политики+тест в этом же PR; политики не рекурсивны (нет вызова функции, читающей ту же таблицу, без definer-роли); definer-функции: владелец rls_definer, search_path='', row_security=off, REVOKE PUBLIC; нет WITH CHECK (true); нет прямых прав app-роли на таблицы токенов; секреты не появляются в логах/тестах; миграция имеет рабочий downgrade.

Privacy: новые данные покрыты consents/retention; аналитика не содержит медицинских значений; экспорт/удаление учитывают новые сущности; embeddings удаляются каскадно.

AI safety (PHASE-08+): ответ невозможен без ai_evidence; retrieval фильтрует по profile_id/pet_id до поиска; фильтр опасных советов включён и протестирован; red-flag тест проходит; human/pet промты и индексы раздельны; внешний LLM за consent-гейтом.

Medical safety: нет автоматических диагнозов; интерпретации маркированы; референсы имеют источник; человеческие нормы недоступны pet-контуру (негативный тест обязателен); никаких демо-данных в реальных профилях.

UX: пустые состояния; статусы confirmed/needs_review/hypothesis различимы; профильный контекст виден; отчёт печатается.

Deployment readiness: runbook обновлён; expected heads в YAML обновлены; rollback описан и реалистичен; smoke-тесты перечислены; мониторинг/алерты для новых фоновых процессов.
