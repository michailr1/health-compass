# HealthMonitor — демо-портал здоровья

Тестовый русскоязычный веб-портал: персональный кабинет для просмотра
генетико-физиологического отчёта. Только UI-демо на обезличенных данных.

## ⚠️ Дисклеймер

- Всё содержимое — **демонстрационные обезличенные данные**. Никаких реальных ФИО,
  исходных VCF, медицинских документов и токенов в проекте нет и быть не должно.
- Портал **не является медицинским изделием** и не предназначен для диагностики
  или назначения лечения.
- Авторизация в этой демо-сборке — **mock**: любые email/пароль принимаются и
  сохраняются только в `localStorage` браузера. Никуда не отправляются.

## Стек

- React 18 + TypeScript + Vite
- TailwindCSS + shadcn/ui
- react-router-dom
- recharts (графики Oura)

## Структура

```
src/
  data/demo.ts          # все моки, API-shaped
  context/AuthContext   # mock-аутентификация
  components/AppLayout  # sidebar (desktop) + bottom nav (mobile)
  pages/                # Login, Dashboard, Oura, Genetics, ActionPlan, Sources, History
```

Данные в `src/data/demo.ts` намеренно оформлены как типизированные объекты,
чтобы легко заменить их на реальные `fetch`/react-query-запросы к приватному backend.

## Что должно быть в production-версии

Эта демо-сборка **не готова к production**. Реальное развёртывание обязано:

1. Использовать **серверную авторизацию** (например, OAuth/OIDC поверх собственного backend
  или managed identity provider) и HTTP-only session cookies. Никакой mock-auth в браузере.
2. Держать данные и логику в **приватном backend** (приватный GitHub-репозиторий и приватный
  сервис). Клиент получает только агрегированные, безопасные для отображения представления.
3. **Не отдавать браузеру исходные медицинские файлы** (VCF, PDF-заключения, сырые экспорты).
  Только сформированные сервером сводки.
4. Хранить любые секреты (ключи API, токены доступа, GitHub PAT) **только на сервере** —
  никогда в коде фронтенда и не в переменных окружения сборки клиента.
5. Логировать доступ, шифровать данные at-rest и in-transit, соблюдать локальные требования
  к обработке медицинских данных.
